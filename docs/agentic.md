# 에이전틱 개발 (Agentic Development)

> AI 에이전트로 소프트웨어를 개발하는 방식과, 에이전트를 애플리케이션에 적용하는 패턴을 정리한다.
> 업계에서 널리 쓰이고 개인적으로도 공감하는 패턴을 모아, 새 작업의 베이스로 삼는다.

이 프로젝트가 다루는 "에이전틱"은 두 층위로 나뉜다.

| 층위 | 무엇을 | 어디에서 드러나나 |
|---|---|---|
| **개발 시간 (dev-time)** | AI 에이전트로 *개발하는* 방식 | 이 문서 + `.claude/` |
| **런타임 (runtime)** | 에이전트를 *애플리케이션에 적용하는* 패턴 | 이 문서 + `backend/src/` (점진 추가) |

---

## 1. 개발 시간 에이전틱 — 에이전트로 개발하기

Claude Code 기반으로 개발할 때의 구성 요소.

### 에이전트 구성

- **단일 에이전트**: 메인 세션. 코드 작성·수정·논의 등 일반 작업.
- **서브에이전트**: 반복 검증 작업을 기준과 함께 미리 정의해 별도 프로세스에서 격리 실행. 메인 세션 컨텍스트를 오염시키지 않고 결과 요약만 받음. 모델 별도 지정 가능(가벼운 작업은 haiku 등)으로 비용·컨텍스트 절약. (`.claude/agents/`)
- **멀티에이전트**: 독립 인스턴스가 병렬 작업 + 상호 메시징. (실험적)
- **오케스트레이션**: 서브에이전트 실행 순서·조건을 정의해 연쇄 실행. 스킬이 서브에이전트를 병렬 호출하는 패턴으로 간이 구현 가능.

### 도구

- **내장 도구**: Read, Edit, Write, Bash, Glob, Grep 등.
- **Skills**: `/명령어`로 호출하는 재사용 프롬프트 패턴. 반복 작업을 기준과 함께 정의. (`.claude/skills/`)
- **Hooks**: 도구 실행 전후에 셸 스크립트를 자동 실행. 린트·테스트·위험 명령 차단 등. (`.claude/settings.json`)
- **MCP**: 외부 시스템을 에이전트의 도구로 연결하는 프로토콜.

### 메모리 / 상태

- **프로젝트 규칙**: `CLAUDE.md` — 항상 로드되는 규칙·맥락.
- **프로젝트 기억**: Auto Memory — 패턴·결정·인사이트 축적.
- **세션 기억**: 대화 컨텍스트 윈도우 (길어지면 자동 압축).
- 구현 상세·아키텍처 결정은 메모리가 아니라 **코드 주석이나 `docs/`** 에 기록한다. 메모리는 컨텍스트를 소비하므로 최소 유지.

### 워크플로

- 기본 패턴: **Plan(계획) → Act(실행) → Observe(관찰) → 반복**.
- **Plan Mode**: 읽기 전용으로 분석 후 계획 수립, 승인 뒤 실행. 큰 작업·방향 잡을 때.

## 2. 런타임 에이전틱 — 애플리케이션 패턴

업계에서 널리 쓰는 패턴을 **"언제 쓰고 언제 안 쓰는가"** 기준으로 정리한다.
(각 패턴: 실행 예제 + 적용/비적용 기준 + 다이어그램 — 점진적으로 채운다.)

### Building Blocks (기초)

- **Augmented LLM**: tool use, retrieval, memory 를 붙인 LLM
- **구조화 출력**: 스키마 강제 응답
- **가드레일**: 입력·출력 검증

### Workflow Patterns (결정적 흐름)

- **Prompt Chaining**: 작업을 단계로 쪼개 순차 처리
- **Routing**: 입력을 분류해 알맞은 경로로
- **Parallelization**: 병렬 분할 또는 투표
- **Orchestrator-Workers**: 오케스트레이터가 동적으로 하위 작업 분배
- **Evaluator-Optimizer**: 생성 → 평가 → 개선 루프

### Agent Patterns (자율적 흐름)

- **ReAct**: 추론(Reasoning) + 행동(Acting) 반복
- **Reflection / Self-critique**: 스스로 점검·수정
- **Multi-agent**: 역할 분담 협업
- **Human-in-the-loop**: 사람 승인 단계 삽입

---

> 상태: 기본 뼈대. 런타임 패턴은 리서치하며 하나씩 예제·기준·다이어그램으로 채운다.
